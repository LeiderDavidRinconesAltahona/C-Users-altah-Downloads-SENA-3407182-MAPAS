# 📌 Git y GitHub – Mapas Conceptuales y Flujos de Trabajo

Este repositorio contiene una colección de **mapas conceptuales y mentales** sobre **Git y GitHub**, diseñados para explicar de forma visual y estructurada los conceptos fundamentales, comandos básicos, repositorios remotos y flujos de trabajo profesionales utilizados en el desarrollo de software.

El material es ideal para **estudiantes**, **principiantes en control de versiones** y **trabajo colaborativo**, así como para apoyo académico y formativo.

---

## 📚 Contenido del repositorio

### 1️⃣ Git y GitHub – Conceptos generales
Mapa conceptual que explica:

- ¿Qué es Git?
- Características principales de Git
- ¿Qué es GitHub?
- Funciones principales de GitHub
- Editores de código y tipos de archivos
- Instalación de Git en:
  - Windows
  - Linux
  - macOS
- Comandos iniciales de Git
- ¿Por qué usar Git?

📌 Este mapa introduce el **control de versiones** y la diferencia entre Git (herramienta) y GitHub (plataforma).

---

### 2️⃣ Comandos básicos de Git y flujo de trabajo
Mapa conceptual enfocado en el uso práctico de Git:

- Staging Area y repositorios
- Ciclo básico de trabajo en Git
- Estados de los archivos:
  - Untracked
  - Unstaged
  - Staged
  - Tracked
- Comandos para mover archivos entre estados
- Eliminación de archivos en Git
- Ramas (Branches):
  - Master/Main
  - Development
  - Hotfix
- Análisis y revisión de cambios:
  - `git log`
  - `git diff`
  - `git show`

📌 Ideal para comprender cómo Git gestiona los cambios paso a paso.

---

### 3️⃣ Repositorios remotos en GitHub
Mapa conceptual sobre el trabajo con repositorios remotos:

- Funciones principales de GitHub
- Archivo `README.md`
- Clonar y conectar repositorios
- Sincronización local ↔ remoto:
  - `git fetch`
  - `git pull`
  - `git push`
- Llaves públicas y privadas
- Configuración de llaves SSH
- Conexión segura a GitHub mediante SSH

📌 Explica cómo trabajar correctamente con repositorios remotos y seguridad.

---

### 4️⃣ Flujos de trabajo profesionales
Mapa conceptual orientado al trabajo colaborativo:

- Flujo de trabajo profesional con Git
- Trabajo con ramas
- Protección de la rama principal (master/main)
- Entornos de trabajo:
  - Develop
  - Staging
  - Producción
- Pull Requests (PR):
  - ¿Qué es un Pull Request?
  - Flujo completo de un PR
  - Revisión de código
  - Aprobación, cambios o rechazo

📌 Representa el **flujo real utilizado en equipos profesionales de desarrollo**.
